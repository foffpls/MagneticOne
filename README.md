## GIS ETL для тестового завдання MagneticOne

Проєкт для виконання тестового завдання на посаду **ГІС розробника**:

- обробка табличних даних (CSV/XLSX);
- підготовка нормалізованої таблиці для точкових геоданих;
- завантаження результату в **Hosted Feature Layer** в **ArcGIS Online** за допомогою **ArcGIS API for Python**.

### Структура проєкту

```
MagneticOne/
├── src/
│   ├── gis_etl/
│   │   ├── __init__.py          # пакет
│   │   ├── config.py            # налаштування (ID шару, URL порталу)
│   │   ├── data_processing.py  # перетворення таблиці (Частина 1)
│   │   └── arcgis_uploader.py   # завантаження в ArcGIS Online (Частина 2)
│   └── telegram_bot/
│       ├── __init__.py          # пакет
│       ├── bot_config.py        # конфігурація бота (токен, посилання на шар)
│       └── bot_app.py           # логіка бота (команди, файли, завантаження)
├── scripts/
│   └── run_bot.py               # запуск Telegram-бота
├── requirements.txt             # залежності Python
├── .env.example                 # приклад змінних оточення
└── README.md
```

### Основні кроки роботи

1. Створити та активувати віртуальне середовище Python.
2. Встановити залежності:
   ```bash
   pip install -r requirements.txt
   ```
3. Заповнити `.env` на основі `.env.example`.
4. Запустити Telegram-бота:
   ```bash
   python -m scripts.run_bot
   ```

### Telegram-бот (aiogram)

Бот обгортає обидві частини завдання і дозволяє працювати через Telegram.

- **Запуск бота**
  1. У `.env` вказати:
     - `TELEGRAM_BOT_TOKEN` – токен бота від `@BotFather`;
     - `ARCGIS_FEATURE_LAYER_URL` – повний URL шару виду `https://.../FeatureServer/0` (обов'язково для завантаження даних у публічний Hosted Feature Layer);
     - `ARCGIS_ITEM_ID` – itemId шару в ArcGIS Online (для посилання на item-сторінку після завантаження);
     - `ARCGIS_PORTAL_URL` – URL порталу (за замовчуванням `https://www.arcgis.com`);
     - `GOOGLE_SHEET_URL` – (необов'язково) посилання на Google Sheets з вихідними даними
       виду `https://docs.google.com/spreadsheets/d/.../edit?gid=0#gid=0` (для команди `/google_spreadsheets`).
  2. Запустити:
     ```bash
     python -m scripts.run_bot
     ```

- **Основні команди бота**
  - `/start` – короткий опис призначення бота та покрокова інструкція:
    - 1️⃣ надіслати файл CSV/XLSX як документ;
    - 2️⃣ отримати оброблений результат (CSV + XLSX);
    - 3️⃣ викликати `/upload_arcgis` для завантаження в Hosted Feature Layer.
  - `/help` – детальний опис можливостей, форматів файлів та контакти розробника.
  - `/google_spreadsheets` – завантажити вихідні дані з Google Sheets (за посиланням у `GOOGLE_SHEET_URL`), обробити їх та отримати CSV/XLSX.
  - `/upload_arcgis` – запуск завантаження останнього обробленого результату в Hosted Feature Layer.

- **Обробка файлів**
  - Надішли боту **файл як документ** (Document) у форматі:
    - `CSV` – з колонками `Дата`, `Область`, `Місто`, `Значення 1–10`, `long`, `lat`;
    - або `XLSX` – з тими ж назвами колонок.
  - Бот:
    - зчитує файл (`pandas.read_csv` або `pandas.read_excel`);
    - перевіряє наявність ключових колонок (`Дата`, `Область`, `Місто`) і, у разі відсутності, повертає зрозуміле повідомлення;
    - перетворює таблицю згідно з алгоритмом з ТЗ (розбиття рядків за значеннями 1–10);
    - рахує коротку статистику (кількість вихідних та вихідних рядків, кількість унікальних міст/областей);
    - повертає **два файли**:
      - оброблений `CSV`,
      - оброблений `XLSX` (для ручної перевірки);
    - запам’ятовує шлях до обробленого CSV для подальшого завантаження в Hosted Feature Layer.

- **Завантаження у Hosted Feature Layer**
  - Команда `/upload_arcgis`:
    - якщо обробленого файлу ще немає – бот просить спочатку надіслати CSV/XLSX;
    - якщо файл є – бот показує попередження:
      - *«УВАГА: будуть видалені всі попередні об'єкти в Hosted Feature Layer та завантажені нові дані з останнього обробленого CSV. Підтверди, будь ласка, цю операцію.»*
    - бот показує **інлайн-клавіатуру**:
      - `✅ Так, замінити дані в шарі` – підтвердження;
      - `❌ Скасувати` – відміна без змін у шарі.
    - при підтвердженні:
      - бот редагує попереднє повідомлення на *«Завантажую дані в ArcGIS Online, зачекай...»*;
      - викликає `upload_dataframe(clear_existing=True)`:
        - очищає шар (`delete_features("1=1")`),
        - додає нові точки (`edit_features(adds=features)`), враховуючи український формат чисел (`long/lat` з комою);
      - у разі успіху:
        - редагує повідомлення на *«Дані успішно завантажено у Hosted Feature Layer. Переглянути шар можна за посиланням: Відкрити шар»* (посилання будується на основі `ARCGIS_ITEM_ID`);
      - у разі помилки:
        - редагує повідомлення на текст з коротким описом помилки (код/опис з ArcGIS).
    - при скасуванні:
      - бот редагує попереднє повідомлення на *«Операцію завантаження скасовано. Дані в Hosted Feature Layer не змінювались.»*

> **Примітка про схему шару:**  
> У тестовому середовищі шар `test_layer_63_zip / test_layer` має такі поля:  
> `date_1`, `Область`, `city`, `value_1..value_10`, `long`, `lat`.  
> Відповідно, у `FIELD_MAPPING` (`gis_etl.arcgis_uploader`) ці назви вже враховані.  
> Якщо використовувати інший Hosted Feature Layer, потрібно оновити `FIELD_MAPPING` під його схему.

### Технологічний стек

- **Python 3.10+** (рекомендовано);
- **pandas** для обробки табличних даних (CSV/XLSX);
- **ArcGIS API for Python** (`arcgis`) для роботи з ArcGIS Online (анонімне підключення до публічного шару);
- **aiogram 3** для Telegram-бота;
- **openpyxl** для читання/запису XLSX.
